<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → template/visit/task/reminder Server Actions | All inputs untrusted; Zod-validated, tenant-scoped, FK-guarded. |
| applyJobTemplate copy | Template content is tenant-owned; the copy must remain within the same tenant. |
| customer detail → /jobs/new prefill params | customerId/contactId/locationId in the URL are validated/guarded by createJob's cross-tenant checks at submit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-02 | Information Disclosure / Elevation | template/visit/task/reminder queries + applyJobTemplate | mitigate | withTenant + RLS + explicit tenant scope; applyJobTemplate reads/writes only the caller's tenant; cross-tenant jobId/templateId guards |
| T-3-01 | Tampering | site-visit status | mitigate | Site-visit status validated against the job_status enum server-side; visits do not bypass enum membership (FSM transition rules apply at the visit-status select level via the shared grouping) |
| T-3-05 | Input Validation | template/visit/task/reminder fields + prefill params | mitigate | Zod safeParse on every action; createJob re-guards the prefilled customerId/contactId/locationId at submit |
| T-3-SC | Tampering | npm/pip/cargo installs | accept | No package installs (reuses installed Base UI + existing settings patterns) |
</threat_model>

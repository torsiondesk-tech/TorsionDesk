<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server Action / lib → Postgres | transitionJobStatus and listJobs run under withTenant; the GUC + RLS enforce tenant isolation. |
| client-supplied toStatus → FSM | Untrusted status string validated against the enum + ALLOWED_TRANSITIONS server-side. |
| numeric strings → money math | rate/cost/qty arrive as strings; integer-cents parsing prevents float corruption. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Tampering | FSM status enforcement | mitigate | transitionJobStatus checks ALLOWED_TRANSITIONS and throws on illegal jumps; status is writable ONLY here; updateJob (Wave 3) excludes status. Wave-0 test asserts the throw. |
| T-3-02 | Information Disclosure | listJobs / getJob / countJobsByTag / transition guard | mitigate | All wrapped in withTenant + explicit `eq(tenantId, orgId)`; transitionJobStatus throws "Job not found" for cross-tenant ids |
| T-3-03 | Tampering | computeJobTotals | mitigate | Server-authoritative integer-cents totals (never trust client totals); inputs are money strings parsed to cents; GP% divide-by-zero guarded |
| T-3-05 | Input Validation | toStatus enum membership | mitigate | Validate toStatus is a JobStatusValue before the map lookup (V5) |
| T-3-SC | Tampering | npm/pip/cargo installs | accept | No package installs; integer-cents needs no decimal lib (RESEARCH Pitfall 2 / A7) |
</threat_model>

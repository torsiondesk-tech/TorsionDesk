<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /jobs RSC query params | bucket/tag/sort are untrusted; listJobs is tenant-scoped and validates filter values. |
| client file upload → uploadJobPhoto | Untrusted binary; MIME/ext allow-list, size cap, fixed tenant path, service-role key server-only. |
| browser bundle ↔ service-role key | The service-role key must never reach the client. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-02 | Information Disclosure | listJobs / getJob / countJobsByTag / equipment query | mitigate | All tenant-scoped via withTenant + RLS; equipment queried by service_location_id (tenant-owned), never cross-tenant |
| T-3-04 | Tampering / DoS | Pics upload (SVG XSS, oversized, path traversal) | mitigate | Image MIME+ext allow-list (reuse resolveExtension), size cap, fixed `${orgId}/jobs/${jobId}/...` path (no client-controlled path), render via Storage URL not inline |
| T-3-06 | Information Disclosure | Service-role key leak to client bundle | mitigate | `import 'server-only'` at top of photos.ts (mirrors storage.ts); upload runs only in a Server Action |
| T-3-EQ | Information Disclosure | Equipment ownership | mitigate | Equipment FK is service_location_id (CLAUDE.md rule); tab never queries by customer_id |
| T-3-SC | Tampering | npm/pip/cargo installs | accept | No package installs (reuses TanStack Table, nuqs, supabase-js already installed) |
</threat_model>

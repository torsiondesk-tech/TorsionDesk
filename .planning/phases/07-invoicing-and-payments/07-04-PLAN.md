<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → recordPaymentAction | Untrusted amount + allocation inputs; server re-validates over-application before insert |
| Admin check → payment method writes | Clerk orgRole checked server-side in createPaymentMethodAction; client cannot bypass |
| Stripe Payment Link clipboard | Clipboard write only on user gesture; URL never exposed to other tenants |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-11 | Tampering | /payments/new over-application | mitigate | Client-side guard is UX; server-side guard in recordPaymentAction is the security control — SUM(allocations) <= amount + 0.001 before any insert |
| T-07-12 | Elevation of Privilege | payment-methods CRUD | mitigate | createPaymentMethodAction, updatePaymentMethodAction, deletePaymentMethodAction all check orgRole === 'admin' via Clerk before withTenant write |
| T-07-13 | Information Disclosure | /invoices/[id] payment link URL | accept | Payment link URL is a non-secret Stripe pay.stripe.com URL; copying it is the intended action; Stripe handles payment completion auth |
| T-07-14 | Tampering | /payments/[id] edit | accept | Edit Payment navigates to /payments/new for v1; no inline mutation from view page; future edit form will add proper input validation |
</threat_model>

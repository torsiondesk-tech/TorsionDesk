<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Clerk → /api/webhooks/clerk | Untrusted POST body crosses here; only a valid Svix signature proves it is from Clerk |
| App server → Supabase Postgres | Tenant scoping must be enforced by RLS, not trusted from the app; service-role key bypasses RLS |
| Browser → middleware/routes | Session token (JWT v2 `o.id`/`o.rol`) is the only tenant/role evidence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-01 | Information Disclosure / Elevation | Cross-tenant read (IDOR at tenant level) | mitigate | RLS policy on `tenants` keyed on `current_setting('app.current_tenant_id', true)::uuid` (fail-closed); all queries routed through `withTenant` |
| T-00-02 | Tampering | Tenant GUC value in `withTenant` | mitigate | `set_config('app.current_tenant_id', ${orgId}, true)` uses a bound parameter — never string-concatenated |
| T-00-03 | Spoofing | Forged Clerk webhook → fake org provisioning | mitigate | `verifyWebhook()` (Svix signature) before any DB write; idempotent insert on replay |
| T-00-04 | Information Disclosure | Supabase service-role key leakage (bypasses RLS) | mitigate | `SUPABASE_SERVICE_ROLE_KEY` documented server-only in `.env.local.example`; never in `NEXT_PUBLIC_*`; used only for provisioning/Storage |
| T-00-05 | Tampering | PgBouncer transaction-pooler prepared-statement misconfig → unscoped queries | mitigate | `postgres(url, { prepare: false })` on the :6543 transaction pooler |
| T-00-SC | Tampering | npm package installs (@clerk/nextjs, drizzle-orm, postgres, svix, zod, etc.) | accept | All verified in RESEARCH Package Legitimacy Audit (slopcheck OK; vitest SUS is a documented false positive). No `[ASSUMED]`/`[SUS]`/`[SLOP]` package requiring a blocking checkpoint |

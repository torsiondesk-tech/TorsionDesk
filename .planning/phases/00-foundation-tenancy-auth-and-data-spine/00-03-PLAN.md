<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → middleware | Session JWT (`o.id`/`o.rol`) is the only role/tenant evidence; middleware is the enforcement point |
| Client nav → routes | Nav hiding is UX only; direct URL access must still be blocked server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-06 | Elevation of Privilege | Dispatcher/Technician reaching `/settings` | mitigate | `auth.protect({ role: 'org:admin' })` at middleware (D-15) — server-side gate, not just nav hiding; sidebar additionally hides Settings for non-admins (defense in depth) |
| T-00-07 | Elevation of Privilege | Technician reaching the web shell | mitigate | Middleware redirects `org:technician` to `/mobile-coming-soon` before any app route renders (D-14) |
| T-00-08 | Spoofing / Information Disclosure | Webhook route accidentally protected → provisioning 401s, or unauth users reaching app | mitigate | `/api/webhooks(.*)` in the public matcher; everything else behind `auth.protect()` (default-public middleware, every route opts in) |
| T-00-09 | Information Disclosure | Relying on client nav hiding alone for access control | mitigate | Nav hiding is declared UX-only (D-16); the middleware role gate (T-00-06) is the enforcement layer |
</threat_model>

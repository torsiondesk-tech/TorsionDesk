## Spacing Scale

Inherited 4-point scale. Declared values (all multiples of 4):

| Token | Value | Usage in Phase 1 |
|-------|-------|------------------|
| xs | 4px | Icon-to-label gaps, badge inner padding, chip gaps in TagSelect |
| sm | 8px | Compact spacing: table cell padding-y, form field label-to-input gap |
| md | 16px | Default spacing: form field vertical rhythm, card padding, drawer content padding |
| lg | 24px | Section padding: tab content padding, two-column form gutter, location section spacing |
| xl | 32px | Layout gaps: page header to content, between location sections in Locations & Equipment tab |
| 2xl | 48px | Major breaks: empty-state vertical padding, top/bottom page padding |
| 3xl | 64px | Reserved — page-level spacing on the full-page New/Edit Customer form max-width container |

Exceptions:
- **Touch targets:** interactive icon-only buttons (table row actions, drawer close, "Add" affordances) render at minimum 40px hit area even when the visual glyph is smaller. This is the only deviation from the 4-scale on size (40 is still a multiple of 4).
- Tailwind utilities map: xs=`gap-1`/`p-1`, sm=`gap-2`/`p-2`, md=`gap-4`/`p-4`, lg=`gap-6`/`p-6`, xl=`gap-8`/`p-8`, 2xl=`gap-12`/`p-12`, 3xl=`gap-16`. Never use arbitrary `[13px]`-style values.

## Typography

Inherited `--font-sans` (system-ui). 4 sizes, 2 weights (regular 400 + semibold 600). No third weight.

| Role | Size | Weight | Line Height | Usage in Phase 1 |
|------|------|--------|-------------|------------------|
| Body | 16px (`text-base`) | 400 | 1.5 | Form field values, table cell text, notes, drawer body copy |
| Label | 14px (`text-sm`) | 400 (600 for column headers) | 1.5 | Field labels, table headers (semibold), helper text, secondary metadata (`text-muted-foreground`) |
| Heading | 20px (`text-xl`) | 600 | 1.2 | Tab section titles ("Doors", "Openers", "Springs"), drawer titles, card titles |
| Display | 28px (`text-3xl` ≈ 30px allowed) | 600 | 1.2 | Customer name in the detail-page header; "New Customer" / "Edit Customer" page title |

- Body line-height: **1.5** (`leading-normal`). Heading/Display line-height: **1.2** (`leading-tight`).
- Numeric/tabular data in the customer list and equipment specs may use `tabular-nums` for column alignment — same size/weight, just the variant.
- Account number on the header renders at Label size, weight 400, `text-muted-foreground` (it is metadata, not the primary identity — the name is).

## Color

The inherited theme is a **near-monochrome neutral palette** (every brand token is `oklch(... 0 0)` — zero chroma grayscale) with a single red destructive. This is intentional for a dense, Service-Fusion-style back-office CRM. The 60/30/10 split is expressed through neutral value steps, with the dark `--primary` acting as the visual accent.

| Role | Token | Value (light) | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `--background` / `bg-background` | `oklch(1 0 0)` (white) | Page canvas, table body, drawer body, form area |
| Secondary (30%) | `--card` / `--secondary` / `--sidebar` | `oklch(1 0 0)` card, `oklch(0.97 0 0)` secondary, `oklch(0.985 0 0)` sidebar | Cards, sidebar nav, tab strip, table header row, drawer chrome, muted section backgrounds |
| Accent (10%) | `--primary` / `bg-primary` | `oklch(0.205 0 0)` (near-black) | See reserved list below |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | Destructive actions ONLY (see copywriting) |

**Accent (`--primary`) reserved for — explicit list, nothing else:**
- The single primary CTA per screen: "Save Customer" on the New/Edit form; "Create New" button in the app header; the primary confirm button inside each drawer ("Add Contact", "Add Location", "Add Equipment", "Save").
- The active tab indicator on the customer-detail tab strip.
- The selected row / selected-field highlight in the Merge comparison view.

Accent is NOT used for: every button (secondary actions use `variant="outline"`/`ghost`), table links (use `text-foreground` + underline-on-hover), badges (VIP/status badges use `secondary`/`outline` variants), or hover states (use `bg-accent` muted token, not `--primary`).

**Status / semantic color usage (Phase 1 specifics):**
- **VIP badge:** `Badge variant="secondary"` with a star icon — neutral, not a new color.
- **Active / Inactive status:** Active = `outline` badge neutral; Inactive/Archived = `muted` badge (`text-muted-foreground`). No green/red status dots in Phase 1 (no new colors introduced).
- **Destructive (red) is reserved for:** Deactivate confirmation, Merge "archive losing record" warning, and field-level delete in drawers (remove contact/location/equipment).
- "Coming in Phase X" placeholder tabs (Jobs, Estimates, Activity-when-empty): muted centered empty state, `text-muted-foreground`, no accent.

## Copywriting Contract

Voice: plain, operational, Service-Fusion-familiar. Verb + noun on every action. No marketing tone — this is an internal tool.

| Element | Copy |
|---------|------|
| Primary CTA (create flow) | **"Save Customer"** (full-page form submit). Header global button: **"Create New"** (dropdown: New Customer / New Job / New Estimate). |
| Drawer CTAs | "Add Contact" / "Add Location" / "Add Equipment" (opens drawer); drawer submit: "Save Contact" / "Save Location" / "Save Equipment". |
| Customer list empty state — heading | **"No customers yet"** |
| Customer list empty state — body | **"Add your first customer to start building your customer book."** with a primary **"Create New" → New Customer** button below. |
| Search no-results state | Heading: **"No matches"** — Body: **"No customers match \"{query}\". Check spelling or search by phone, email, or address."** |
| Contacts tab empty state | **"No contacts on this customer."** + "Add Contact" button. |
| Locations tab empty state | **"No service locations yet."** + "Add Location" button. (Equipment Add buttons appear only inside a location section.) |
| Equipment section empty (per type) | Inline muted line: **"No doors recorded."** / **"No openers recorded."** / **"No springs recorded."** each with its own "Add" button. |
| Activity tab empty state | Heading: **"No activity yet"** — Body: **"Jobs, estimates, invoices, and emails for this customer will appear here as they happen."** (This is the live empty state for the pre-built `customer_events` feed, NOT a "coming soon" stub.) |
| Jobs / Estimates stub tabs | Heading: **"Jobs coming in a later release"** / **"Estimates coming in a later release"** — Body: muted single line. (Matches Phase 0 stub-tab pattern; no fake data.) |
| Form validation error (field) | Inline below field, red text, specific: e.g. **"Name is required."**, **"Width must be a positive number."**, **"Wire size must be a decimal in inches."** (Mirrors the Zod messages in RESEARCH.) |
| Server action error (banner) | **"Couldn't save. {reason}. Try again."** — never leak DB internals or other-tenant data. |
| Account number helper | On the form: **"Assigned automatically when you save."** (read-only field, since account_no is `max+1` server-side). |
| Type-ahead placeholder | Customer search: **"Search by name, phone, email, or address…"** Parent account: **"Search for a parent account…"** |
| TagSelect / ReferralSelect create-option | **"Create \"{typed value}\""** (e.g. `Create "SPRINGS"`). |

**Destructive actions in Phase 1 (each requires confirmation):**

| Destructive action | Confirmation approach |
|--------------------|----------------------|
| **Deactivate customer** | Base UI `Dialog` (AlertDialog pattern). Title: **"Deactivate this customer?"** Body: **"They'll be hidden from the active list but their records are kept. You can reactivate them later."** Confirm button: destructive variant, label **"Deactivate"**. Cancel is default. |
| **Merge customers (archive loser)** | Confirmation step inside the side-by-side merge flow. Title: **"Merge into one record?"** Body: **"\"{loser}\" will be archived and all its contacts, locations, and equipment move to \"{winner}\". This can't be auto-undone."** Confirm: destructive variant, label **"Merge & Archive"**. |
| **Remove contact / location / equipment** (in-drawer or row action) | Base UI `Dialog`. Title: **"Remove this {item}?"** Body: **"This permanently removes the {item} from this customer."** Confirm: destructive, label **"Remove"**. |

No bare browser `confirm()`. All confirmations use the Base UI `Dialog` component.

## Interaction & Layout Contracts (Phase-1 specific)

These lock the SF-parity decisions from CONTEXT (D-01..D-20) into concrete UI behavior the executor implements.

| Surface | Contract |
|---------|----------|
| Customer detail header (D-01, D-05) | Top band: customer name (Display), account # + VIP badge + status badge (Label/metadata), action bar right-aligned: `Edit Customer` (outline) · `New Job`/`New Estimate` (outline, disabled stub w/ tooltip "Available in a later release") · `Merge` (outline) · `Deactivate` (ghost, destructive on confirm). Below: tab strip. |
| Tab strip (D-01) | Base UI `Tabs`: Summary · Contacts · Locations & Equipment · Jobs (stub) · Estimates (stub) · Activity · Notes. Active tab uses `--primary` indicator. Stub tabs render the "coming in a later release" empty state. |
| Customer list (D-04, CUST-09) | TanStack Table v8, `manual*` flags, server-side. Columns: Name (link) · Account # · Phone · City · Tags (badges) · Status. Toolbar: search input (type-ahead, 250ms debounce, placeholder above), status filter, city filter, tag filter — all synced to URL via `nuqs`. Sticky header row uses secondary surface. Row click navigates to detail. |
| Add drawers (D-09, D-13) | Base UI `Sheet`, slides from right. Width ≈ 480px desktop, full-width on mobile. Header (drawer title, semibold xl) · scrollable body (form, md vertical rhythm) · sticky footer (Cancel ghost + primary Save). Equipment drawer: Type `select` at top → conditional fields reveal below (D-13). |
| Equipment grouping (D-02) | Locations & Equipment tab: each service location is a `Card` section (xl gap between locations). Inside each: three labeled subsections — Doors / Openers / Springs (Heading size) — each listing records and its own "Add" button. |
| Typed inputs (D-15) | Wind direction = Base UI `radio-group` styled as segmented control (Left / Right / Pair). Door size = two number inputs with unit suffix labels (`W [ ] ft × H [ ] ft`). Wire size + inside diameter = decimal number inputs with "in" unit label. No free-text for these. |
| Creatable selects (D-17, D-18) | `<TagSelect>` and `<ReferralSelect>` = Base UI `Combobox` (`multiple` + chips for tags). Inject `Create "{value}"` option on no-match. Shared components — built to be imported by Phases 3/6. |
| Merge view (D-20) | Two-column side-by-side `Card`s with per-field radio selection (pick winner value per field). Selected field highlighted with `--primary` ring. Confirm = destructive dialog above. |
| Responsiveness | Mobile-first. List table collapses to stacked cards under `md`. Two-column customer form stacks to single column under `md`. Drawers go full-width under `sm`. |

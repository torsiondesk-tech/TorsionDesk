<objective>
Deliver the admin-facing Settings > Email and Settings > SMS panels (COMM-02, COMM-08), activate the two stub tabs, backfill communication_triggers + communication_settings for existing tenants (and seed new ones via provisionTenant), and rewire the INV-12 "Email Opened" read to derive from communication_logs (D-13) instead of the invoices.emailOpenedAt column. This is the frontend configuration layer plus the one read-side enhancement.

Purpose: COMM-02/COMM-08 require the settings panels that gate every send; without the backfill, the single dev tenant has no trigger rows and every send would be skipped by the enabled-gate. The INV-12 rewire makes the webhook generic (updates communication_logs only, per D-13) while the invoice UI still shows open tracking.
Output: Working Email/SMS settings, seeded/backfilled tenants, communication_logs-derived Email Opened.

Sequencing note: this plan touches src/app/(app)/communications/actions.ts (created in 08-02) and src/app/(app)/invoices/actions.ts (also modified by 08-03). It therefore runs in Wave 4, after 08-03, to avoid a file-ownership conflict on invoices/actions.ts. It shares Wave 3/4 boundaries with 08-04 but has zero file overlap with it.
</objective>

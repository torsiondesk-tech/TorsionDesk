# Phase 6 ‚Äî UI Design Contract

> Visual and interaction contract for the Estimates module. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Pre-populated from the detected design system (`components.json`, `src/app/globals.css`, root `layout.tsx`) and `06-CONTEXT.md`. ## Typography

Geist sans, sizes/weights mapped to existing Tailwind utilities. Exactly 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Display (page title ‚Äî "Estimates", "Estimate #EST-0042") | 28px | 600 semibold | 1.2 | `text-3xl font-semibold` |
| Heading (panel titles "Project Specs" / "Sales Data", group header rows, card titles) | 20px | 600 semibold | 1.2 | `text-xl font-semibold` |
| Body (form values, line-item cells, dashboard rows) | 16px | 400 regular | 1.5 | `text-base` |
| Label (field labels, table column headers, status badge text, muted meta) | 14px | 400 regular (600 for column headers) | 1.5 | `text-sm` / `text-sm font-semibold` |

Body line height 1.5; heading/display line height 1.2. Muted secondary text uses `text-muted-foreground` at the Label size. No third weight ‚Äî emphasis comes from size + `text-muted-foreground`, not extra weights.

## Color

Neutral OKLCH system (already defined in `globals.css`). The 60/30/10 split below describes how the Estimates surfaces should distribute color.

| Role | Value (token) | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `oklch(1 0 0)` / dark `oklch(0.145 0 0)` ‚Üí `bg-background` | Page background, form panel surfaces, main estimate list canvas |
| Secondary (30%) | `--card` / `--sidebar` / `--muted` (`oklch(0.97‚Äì0.985 0 0)`) ‚Üí `bg-card`, `bg-sidebar`, `bg-muted` | Status-folder sidebar, line-item group header rows, totals card, table header row |
| Accent (10%) | `--primary` `oklch(0.205 0 0)` ‚Üí `bg-primary text-primary-foreground` | See reserved list below |
| Destructive | `--destructive` `oklch(0.577 0.245 27.325)` ‚Üí `text-destructive` / `variant="destructive"` | "Lost" status emphasis and the delete-estimate confirmation only |

**Accent (`primary`) reserved for ‚Äî explicit list:**
- The single primary CTA per screen: "New Estimate" (dashboard) and "Save Estimate" (form)
- "Convert to Job" action button on the estimate detail header (primary emphasis ‚Äî it is the phase's headline action)
- The active/selected status folder in the sidebar (selected-state highlight)
- Filled stars in the opportunity-rating star picker (filled = `text-primary` or `fill-primary`, empty = `text-muted-foreground`)

Accent is NEVER applied to all interactive elements. Secondary buttons ("Download PDF", "Email Estimate" stub, "Add Group", "Add Line Item", "Cancel") use `variant="outline"` or `variant="secondary"`. Status badges follow the existing `statusBadgeVariant` convention from `src/lib/jobs/transitions.ts` ‚Äî map estimate statuses as: Requested/Provided ‚Üí `outline`, Accepted ‚Üí `default`, Won ‚Üí `secondary`, Lost ‚Üí `destructive`. Add an `estimateStatusBadgeVariant` helper mirroring the jobs one rather than inventing new colors.

# Phase 4 — UI Design Contract

> Visual and interaction contract for the Dispatch Board. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Service Fusion parity is a hard requirement: layout, terminology, and popup/pool labels must feel near-identical to SF.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (detected in `components.json`) |
| Preset | base-nova (style); baseColor neutral; cssVariables true |
| Component library | base-ui (Base UI — use the `render` prop, NEVER `asChild`; bare DialogTrigger+Button nests buttons) |
| Icon library | lucide (lucide-react) |
| Font | system-ui / -apple-system, var(--font-sans) — set in `globals.css` |

Tokens are already declared in `src/app/globals.css` via Tailwind v4 `@theme inline` (oklch CSS variables, light + dark). This phase MUST consume the existing variables (`bg-background`, `bg-card`, `bg-sidebar`, `text-muted-foreground`, `border`, `bg-secondary`, etc.) — do NOT introduce new raw hex values. Radius scale (`--radius` 0.625rem → `rounded-md`/`rounded-lg`/`rounded-2xl`) is also pre-declared.

Existing reusable UI primitives (do not rebuild): button, card, badge, dialog, sheet, popover, select, combobox, tabs, table, separator, input, textarea, label, checkbox, input-group.

## Spacing Scale

Declared values (multiples of 4 — matches the 8-point scale already used across Phases 1–3):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps inside a job block, badge inset (`gap-1`, `px-1`) |
| sm | 8px | Job-block internal padding, stacked-block vertical gap within a day cell (`p-2`, `gap-2`) |
| md | 16px | Default gap between grid toolbar controls, popup section spacing (`gap-4`, `p-4`) |
| lg | 24px | Dispatch popup column padding, board section padding (`p-6`) |
| xl | 32px | Gap between the calendar grid and the job pool below it (`gap-8` / `pt-8`) |
| 2xl | 48px | Major page section breaks (header → board) |
| 3xl | 64px | Page-level top/bottom breathing room |

Exceptions:
- **Day-column min width:** 160px each (7 columns) — a layout dimension, not a spacing token; chosen for legible 5-field blocks at common 1280–1920px widths.
- **Tech-name (first) column width:** 180px fixed, sticky-left.
- **Day-header row height:** 44px (sticky-top) — meets the 44px touch-target minimum for the Today/chevron controls.
- **Job block min height:** 72px so all five fields (D-06) render without clipping; cell grows to fit stacked blocks (D-07).

## Typography

3 roles, 2 weights (regular 400 + semibold 600). No new font; uses `var(--font-sans)`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Block detail (customer name in block, popup field labels, pool meta) | 14px (`text-sm`) | 600 semibold for the primary line, 400 regular for meta | 1.5 |
| Body (popup field values, addresses, descriptions) | 16px (`text-base`) | 400 regular | 1.5 |
| Heading (popup customer name, board page title, pool tab labels) | 20px (`text-xl`) | 600 semibold | 1.2 |
| Micro (job#, arrival-time window, status badge text, count badges) | 12px (`text-xs`) | 600 semibold | 1.2 |

Notes:
- Job# (`#JOB-1042`) renders in `text-xs` semibold, muted (`text-muted-foreground`) so the customer name (`text-sm` semibold foreground) is the visual anchor of each block (D-06).
- Arrival-time window (`8am–12pm`) uses `text-xs` regular, muted.
- All five block fields truncate with `truncate` / `overflow-hidden` when the block is narrow (D-06).

## Color

Built entirely on existing oklch CSS variables — no new palette. 60/30/10 split:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch(1 0 0) light / oklch(0.145 0 0) dark) | Board canvas, grid cells, page surface |
| Secondary (30%) | `--sidebar` + `--card` + `--secondary` (oklch(0.985/0.97/0.205…)) | Sticky tech-name column, day-header row, job pool container, popup card surface |
| Accent (10%) | `--primary` (oklch(0.205 0 0) light) | Primary action button only (see reserved list) |
| Destructive | `--destructive` (oklch(0.577 0.245 27.325)) | "Cancel" job action + its confirmation dialog only |

Accent reserved for:
- The single primary popup action button: **"Close & Invoice"** (the phase's primary CTA) rendered as `variant="default"`.
- The active job-pool tab indicator.
- The "Today" button when the current week is in view.
All other popup actions are `variant="outline"` or `variant="ghost"`. Accent is NEVER applied to all interactive elements.

### Job-block status colors (DISP-04 / D-05)

Color-coded by status **group** (from existing `STATUS_GROUPS` in `src/lib/jobs/transitions.ts`). Use Tailwind utility tints (the codebase already uses `amber-50/700/200`-style tinted badges in `job-form.tsx`, so these match house style) layered over the neutral grid — they are functional status semaphores, counted separately from the 10% brand accent:

| Status group (members) | Block background / border / text |
|------------------------|----------------------------------|
| Open (`unscheduled`, `scheduled`, `dispatched`) | `bg-slate-50 border-slate-200 text-slate-700` (dark: `bg-slate-800/40 border-slate-700 text-slate-200`) |
| In Progress (`delayed`, `on_the_way`, `on_site`, `started`, `paused`, `resumed`, `partially_completed`, `completed`) | `bg-blue-50 border-blue-200 text-blue-700` (dark: `bg-blue-900/30 border-blue-800 text-blue-200`) |
| Closed (`invoiced`, `paid_in_full`, `job_closed`) | `bg-green-50 border-green-200 text-green-700` (dark: `bg-green-900/30 border-green-800 text-green-200`) |
| Cancelled (`cancelled`) | `bg-muted text-muted-foreground opacity-60` — dimmed/muted, struck-through job# optional |

Status **badge** inside the block reuses the existing `statusBadgeVariant()` + `statusLabel()` helpers from `transitions.ts` — do not invent a parallel badge mapping.

## Copywriting Contract

Service Fusion parity: popup action names and pool tab names are FIXED strings and must match SF exactly.

| Element | Copy |
|---------|------|
| Primary CTA | **Close & Invoice** (transitions job → Invoiced; no invoice record yet — Phase 7) |
| Empty state heading (whole board, no jobs this week) | **No jobs scheduled this week** |
| Empty state body (board) | Drag a job from the pool below onto a technician's day to schedule it. |
| Empty state heading (a pool tab with 0 jobs) | **Nothing here** |
| Empty state body (pool tab) | No jobs in {tab name} right now. |
| Empty state (no active technicians) | **No active technicians.** Add a technician under Settings → Users to start dispatching. |
| Error state (Realtime subscription dropped) | **Live updates paused — reconnecting…** Changes will catch up automatically. (auto-retry; show a small inline banner, not a blocking modal) |
| Error state (optimistic drop rejected by server) | **Couldn't move that job.** It's been put back. Try again. (toast; block snaps to original cell per D-10) |
| Error state (board fetch failed) | **Couldn't load the board.** Check your connection and retry. + Retry button |
| Destructive confirmation | **Cancel job** — "Cancel job #{job#}? This sets the job to Cancelled. You can re-open it later." Confirm button label: **Cancel Job** (destructive); dismiss label: **Keep Job** |

Fixed SF-parity labels (must not be reworded):

- **Popup action buttons (D-13):** `Dispatch`, `View Details`, `Make Changes`, `Deposits`, `Close & Invoice`, `Cancel`, `Email`, `Print`.
  - `Deposits` — disabled, tooltip/label suffix "(coming soon)" (Phase 7).
  - `Email`, `Print` — disabled, "(coming soon)" (Phase 8+).
- **Job pool tabs (D-25):** `Unscheduled`, `Unassigned`, `With Open POs`, `Partially Completed`, `Paused`, `Marked For Follow Up` — each with a live count badge.
- **Week navigation (D-04):** `Today` button; Prev/Next chevrons; clicking the current date-range label opens a popover date picker.

Map fallback (D-15, null lat/lng): gray placeholder tile, full address centered as text, button labeled **Open in Maps** (opens `https://maps.google.com/?q={address}` in a new tab). No on-the-fly geocoding.
